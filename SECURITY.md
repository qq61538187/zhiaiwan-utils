# Security Policy

Thank you for helping improve the security of `@zhiaiwan/utils`.

## Supported Versions

Only the latest published major/minor line is actively supported with security fixes.

| Version | Supported |
|---|---|
| Latest | :white_check_mark: |
| Older versions | :x: |

## Reporting a Vulnerability

Please do **not** open public GitHub issues for suspected vulnerabilities.

Use one of the private channels below:

- Open a private GitHub security advisory (preferred)
- Email: `61538187@qq.com`

Please include:

- A clear vulnerability description
- Affected API/method(s)
- Reproduction steps or proof of concept
- Expected impact and attack preconditions
- Suggested remediation (if available)

## Response Process

- **Initial response target:** within 72 hours
- **Triage decision target:** within 7 days
- **Fix and coordinated disclosure:** based on severity and patch readiness

## Severity Targets (SLA)

- **Critical/High:** target patch plan within 7 days after triage
- **Medium:** target patch plan within 30 days after triage
- **Low:** best effort in regular maintenance cycles

We may request additional details during triage. Credit can be provided in release notes unless you prefer to remain anonymous.
